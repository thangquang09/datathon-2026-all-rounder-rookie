# Data Split and Cross-Validation

This package uses time-aware validation only. Random K-Fold is not used because
daily forecasting rows are temporally dependent and future target values must
not influence earlier forecast cutoffs.

## Forecast Task

- Train target file: `data/sales.csv`
- Train target period: `2012-07-04` to `2022-12-31`
- Forecast period: `2023-01-01` to `2024-07-01`
- Forecast horizon: 548 daily rows
- Targets: `Revenue`, `COGS`

`sample_submission.csv` is read only for the `Date` column where needed to
preserve the required row order. `Revenue` and `COGS` values from sample/test
files are never loaded as features.

## Leakage-Safe Feature Rules

- Calendar and holiday features are deterministic transforms of `Date`.
- Vietnamese lunar events are generated by solar-to-lunar conversion from the
  Gregorian date, not from a year-specific external holiday table.
- Future operational aggregates are not used. Rows after the training cutoff are
  blanked or replaced by train-cutoff climatology.
- Legacy `v1` excludes same-day target-proxy operational columns and their
  short-lag/rolling derivatives from the training feature list.
- Target lags in forecast periods use only values known at the cutoff or prior
  recursive predictions.
- Day-of-year, month, weekday, and month-weekday priors are computed only from
  history available up to each cutoff.

## Recursive Pipeline CV

Implemented in `model_thang/forecast_pipeline.py`.

This validates the recursive ensemble and simple seasonal baselines on two
548-day backtests:

| Fold | Train Cutoff | Validation Window |
| --- | --- | --- |
| `fold_2020_2021` | `2020-06-30` | `2020-07-01` to `2021-12-30` |
| `fold_2021_2022` | `2021-06-30` | `2021-07-01` to `2022-12-30` |

Models/components evaluated:

- `model`: recursive LightGBM ensemble
- `seasonal`: seasonal naive / lag-based recurrence
- `doy`: day-of-year climatology
- `cv_weighted_ensemble`: CV-tuned blend of the three components

Artifacts:

- `model_thang/artifacts/cv_metrics.csv`: fold-level metrics.
- `model_thang/artifacts/cv_weights.json`: tuned component weights.
- `model_thang/artifacts/cv_metrics_mean_std.csv`: mean/std summary by
  target and model.

## Direct Factory CV

Implemented in `model_thang/explainable_forecast_factory.py`.

The direct model creates pseudo-forecast rows: each training example represents
“forecast date D from cutoff C” with `horizon = D - C`. This makes training
closer to the real 548-day forecast task than one-step-ahead random rows.

Training cutoffs used to construct direct rows:

```text
2014-12-31, 2015-06-30, 2015-12-31, 2016-06-30,
2016-12-31, 2017-06-30, 2017-12-31, 2018-06-30,
2018-12-31, 2019-06-30, 2019-12-31, 2020-06-30,
2020-12-31, 2021-06-30, 2021-07-01
```

Validation cutoffs:

```text
2020-06-30, 2020-12-31, 2021-07-01
```

Models/components evaluated:

- `lgb`: direct LightGBM model.
- `ridge`: ridge regression pipeline on the same feature rows.
- `doy_prior`: train-cutoff day-of-year prior.
- `weighted_direct`: out-of-fold weighted blend aggregate. This row is useful
  for aggregate model selection but is excluded from fold-level std summaries.

Artifacts:

- `model_thang/artifacts/direct_factory_cv_metrics.csv`: cutoff-level metrics.
- `model_thang/artifacts/direct_factory_cv_metrics_mean_std.csv`: mean/std
  summary by target and model, excluding `fold_cutoff = all_oof`.
- `model_thang/artifacts/direct_factory_audit.json`: CV, model files, leakage
  policy, and holiday source audit.

## Baseline Comparison

`scripts/run_baselines.py` writes:

- `docs/tables/baseline_results.csv`: single-split baseline model comparison.
- `docs/tables/pipeline_results.csv`: fold-level pipeline component comparison.

Because `baseline_results.csv` is single-split, its combined summary has
`n_folds = 1` and blank `*_std` columns.

## Combined Mean/Std Summary

For reporting, use:

```text
model_thang/artifacts/model_baseline_metrics_mean_std.csv
```

Columns:

```text
source,target,model,n_folds,mae_mean,mae_std,rmse_mean,rmse_std,r2_mean,r2_std
```

This file combines recursive CV, direct factory CV, and single-split baselines
in one machine-readable table.
